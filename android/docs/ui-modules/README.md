# UI Module Documentation

This directory documents the Android presentation layer in the Air Mouse app.

## Goals
- Explain what each UI folder is responsible for.
- Help reviewers find the correct screen, state, and view model quickly.
- Describe the main demo flow: register user, calibrate sensors, pair to desktop, then use mouse controls.

## Main Flow
1. Open the app.
2. Register a user name if prompted.
3. Complete calibration.
4. Connect to the desktop server.
5. Use the home hub to reach touchpad, logs, gesture studio, settings, and other modules.
